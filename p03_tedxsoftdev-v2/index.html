Node.js


by Kazi Jamal and Ahmed Sultan, pd 9

What is Node.js?

Node.js is a JavaScript runtime platform, based off Google Chrome's V8 JavaScript engine

It is...

  • OPEN SOURCE
  • SCALABLE
  • SUPPORTED ACROSS PLATFORMS

What are Node.js' uses?

There are two main ways people today use Node.js:

  1. Creating utilities to be used locally, or on servers
  2. Creating web servers and applications
    • Node.js attempts to optimize processing speeds and scalability; strong choice for the development of web applications
    • Cornerstone of some modern-day 'web development stacks' (MEAN, MERN, etc.)
    • Lots of big-name corporations have shifted toward Node.js (Microsoft, Netflix, LinkedIn, PayPal, Uber, etc.)

How do we use Node.js?

Unlike the JavaScript many of us have experienced in SoftDev—whose runtime is confined entirely within the browser—Node.js gives your JavaScript code an environment to run on your computer.

  • Put simply: with Node.js, you can run your JavaScript code in a terminal/locally

Okay... what's npm?

npm is a package manager for JavaScript—and it is the default package manager for Node.js

You can install packages from npm to use in your project

  • by using the command npm init, we can initialize a package which includes a package.json—which, in turn, stores dependencies (modules you have already installed) and other metadata about your project/package

Popular npm packages

  • express: "fast, unopinionated, minimalist web framework for Node.js"
  • lodash: utility library, adding methods to iterate through arrays, strings, etc.
  • react: a library for building user interfaces

Why use Node.js?

Advantages of Node.js

  • Allows programmers to write both the front-end and back-end of web applications in a single language
  • Very large community—lots of learning resources and npm modules to utilize
  • High performance, with Google's V8 JavaScript engine
  • Can process many requests concurrently, better than Ruby or Python
  • Lightweight and scalable

Disadvantages of Node.js

  • Lacking in performance for heavy computational and CPU bound tasks
  • Lacking in maturity—the Node.js API is frequently updated and modified
  • Some npm modules lack thorough documentation

Node.js vs. Flask/Python

  • Python primarily uses pip for library management... while Node.js uses npm
  • Python comes with a built-in environment manager... while Node.js requires you to install one (for example: nodeenv) separately.
  • Flask often has more mature and stable libraries with solid documentation... while Node.js modules sometimes lack thorough documentation.

Demo: a basic web server with Node.js

Creating a basic web server is a great way to introduce yourself to some core Node.js concepts such as modules, request objects and response objects.

	      
var http = require('http');

var requestHandler = function(request, response) {
    console.log(`received a request for ${request.url}`);
    response.write("hello world");
    response.end();
};

var webServer = http.createServer(requestHandler);

webServer.listen(3000, console.log('listening on port 3000'));

This example works as follows:

  1. The core Node.js http module is imported.
  2. A request handler function is created. This function takes a request, logs the request's URL to the console, and then writes "hello world" into the response.
  3. Using the http module's createServer function, a server is created with the previously declared request handler function.
  4. The web server is directed to listen for requests on port 3000.

Opening up a shell in the directory where we put this code in a file called webserver.js, we can test our web server.

	      
devos@softdev:~/nodejs $ ls
webserver.js
devos@softdev:~/nodejs $ node webserver.js
listening on port 3000

Typing in the command node with our file webserver.js starts the server, which prints: listening on port 3000. Now we can try sending a request to the web server by visiting the address localhost:3000 in a web browser.

Upon sending a request through a web browser, the web browser displays "hello world". Looking back in our shell, we see the following.

	      
devos@softdev:~/nodejs $ node webserver.js
listening on port 3000
received a request for /

The web server can be stopped by quitting the process using ctrl + c.

Creating your own modules and importing them for use with Node.js

Modules are a set of functions or variables that are used to organize codebases by separating its different parts.

Node.js comes with some built in modules, like http which we imported in the last example, but you can also create your own.

	      
var x = 8;

var foo = function() {
    console.log('foo is a function!');
};

module.exports.x = x;
module.exports.foo = foo;

After declaring the variable x and the function foo in this file, they are exported using module.exports, in order to be used in other files.

Below is an example of a file that imports the module we just created in order to use its functions and variables.

	      
var demomodule = require('./demomodule')

var x = demomodule.x;
var foo = demomodule.foo;

console.log(x);
foo();

In this file, we import the previously created module named demomodule using the require function. This new demomodule variable is a regular JavaScript object, and its functions and variables can now be utilized within this file.

Opening up a shell in the directory where we put these two files, we can execute our code.

	      
devos@softdev:~/nodejs $ ls
demomodule.js import.js
devos@softdev:~/nodejs $ node import.js
8
foo is a function
devos@softdev:~/nodejs $
